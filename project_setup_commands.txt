HOSPITAL MANAGEMENT SYSTEM - PROJECT SETUP COMMANDS
====================================================

This document contains all the commands used to create a complete hospital management system
with Django backend and React frontend, along with explanations for each command.

================================================================================
BACKEND SETUP (Django)
================================================================================

1. NAVIGATE TO BACKEND DIRECTORY
   Command: cd backend
   Explanation: Changes the current directory to the backend folder where Django project will be created.

2. CREATE VIRTUAL ENVIRONMENT
   Command: python -m venv venv
   Explanation: Creates a Python virtual environment named 'venv'. This isolates project dependencies
   from the system Python installation, preventing conflicts between different projects.

3. ACTIVATE VIRTUAL ENVIRONMENT (Windows PowerShell)
   Command: venv\Scripts\Activate.ps1
   Explanation: Activates the virtual environment. This changes the Python interpreter to use
   the one in the venv folder, ensuring all packages are installed in the project's isolated environment.

4. ACTIVATE VIRTUAL ENVIRONMENT (Windows Command Prompt)
   Command: venv\Scripts\activate.bat
   Explanation: Alternative activation command for Windows Command Prompt users.

5. INSTALL DJANGO
   Command: pip install django
   Explanation: Installs Django web framework, which provides the foundation for building web applications
   with Python.

6. INSTALL DJANGO REST FRAMEWORK
   Command: pip install djangorestframework
   Explanation: Installs Django REST Framework, which provides tools for building REST APIs
   that can communicate with the React frontend.

7. INSTALL CORS HEADERS
   Command: pip install django-cors-headers
   Explanation: Installs CORS (Cross-Origin Resource Sharing) headers package, which allows
   the Django backend to accept requests from the React frontend running on a different port.

8. INSTALL POSTGRESQL ADAPTER
   Command: pip install psycopg2-binary
   Explanation: Installs PostgreSQL database adapter for Python, allowing Django to connect
   to PostgreSQL database (alternative to SQLite).

9. INSTALL ENVIRONMENT VARIABLES PACKAGE
   Command: pip install python-decouple
   Explanation: Installs package for managing environment variables and configuration settings
   securely (keeps sensitive data like database passwords out of code).

10. CREATE DJANGO PROJECT
    Command: django-admin startproject hospital_management .
    Explanation: Creates a new Django project named 'hospital_management'. The dot (.) at the end
    tells Django to create the project in the current directory instead of creating a subdirectory.

11. CREATE DJANGO APPS
    Commands:
    python manage.py startapp patients
    python manage.py startapp doctors
    python manage.py startapp appointments
    python manage.py startapp departments
    python manage.py startapp staff
    Explanation: Creates Django applications for different modules of the hospital system.
    Each app handles a specific functionality (patients, doctors, appointments, etc.).

12. CREATE REQUIREMENTS FILE
    Command: pip freeze > requirements.txt
    Explanation: Saves all installed packages and their versions to requirements.txt file.
    This allows others to install the exact same dependencies using 'pip install -r requirements.txt'.

13. RUN INITIAL MIGRATIONS
    Commands:
    python manage.py makemigrations
    python manage.py migrate
    Explanation: 
    - makemigrations: Creates database migration files based on model changes
    - migrate: Applies the migrations to create/update database tables

14. CREATE SUPERUSER
    Command: python manage.py createsuperuser
    Explanation: Creates an admin user account that can access Django admin interface
    to manage the application data.

15. RUN DEVELOPMENT SERVER
    Command: python manage.py runserver
    Explanation: Starts the Django development server, typically on http://127.0.0.1:8000/
    This allows you to test the backend API.

================================================================================
FRONTEND SETUP (React)
================================================================================

16. NAVIGATE TO FRONTEND DIRECTORY
    Command: cd frontend
    Explanation: Changes directory to the frontend folder where React application will be created.

17. CREATE REACT APP WITH TYPESCRIPT
    Command: npx create-react-app . --template typescript
    Explanation: Creates a new React application in the current directory with TypeScript support.
    TypeScript provides type safety and better development experience.

18. INSTALL MATERIAL-UI COMPONENTS
    Commands:
    npm install @mui/material @emotion/react @emotion/styled
    npm install @mui/icons-material
    Explanation: Installs Material-UI (MUI) library which provides pre-built, customizable
    React components following Google's Material Design guidelines.

19. INSTALL MUI DATA COMPONENTS
    Commands:
    npm install @mui/x-data-grid
    npm install @mui/x-date-pickers
    npm install @mui/x-charts
    Explanation: Installs additional MUI components for:
    - Data Grid: Display data in table format
    - Date Pickers: Select dates for appointments
    - Charts: Create visualizations and reports

20. INSTALL ROUTING
    Command: npm install react-router-dom
    Explanation: Installs React Router for navigation between different pages/components
    in the single-page application.

21. INSTALL HTTP CLIENT
    Command: npm install axios
    Explanation: Installs Axios, a promise-based HTTP client for making API calls
    to the Django backend.

22. INSTALL DATA FETCHING LIBRARY
    Command: npm install react-query
    Explanation: Installs React Query for efficient data fetching, caching, and
    synchronization with server state.

23. INSTALL TYPE DEFINITIONS
    Command: npm install @types/react-router-dom
    Explanation: Installs TypeScript type definitions for React Router, providing
    better IntelliSense and type checking.

24. INSTALL DATE UTILITIES
    Command: npm install date-fns
    Explanation: Installs date-fns library for date manipulation and formatting
    in JavaScript.

25. INSTALL FORM HANDLING
    Commands:
    npm install react-hook-form
    npm install @hookform/resolvers yup
    Explanation: Installs React Hook Form for efficient form handling and Yup for
    form validation schemas.

26. INSTALL DEVELOPMENT DEPENDENCIES
    Commands:
    npm install --save-dev @types/node
    npm install --save-dev prettier
    npm install --save-dev eslint-config-prettier
    Explanation: Installs development tools:
    - @types/node: TypeScript definitions for Node.js
    - prettier: Code formatter
    - eslint-config-prettier: ESLint configuration for Prettier

27. START REACT DEVELOPMENT SERVER
    Command: npm start
    Explanation: Starts the React development server, typically on http://localhost:3000/
    This allows you to view and test the frontend application.

GIT VERSION CONTROL SETUP
================================================================================

28. NAVIGATE TO PROJECT ROOT
    Command: cd /c/Users/anubh/OneDrive/Desktop/hospital
    Explanation: Returns to the root directory of the project to initialize git repository.

29. INITIALIZE GIT REPOSITORY
    Command: git init
    Explanation: Initializes a new Git repository in the current directory, creating
    a .git folder to track changes.

30. ADD REMOTE REPOSITORY
    Command: git remote add origin https://github.com/ry538663/Hospital.git
    Explanation: Connects the local git repository to the remote GitHub repository
    where the code will be stored.

31. CREATE .GITIGNORE FILE
    Command: echo "# Python, Django, Node.js, React exclusions..." > .gitignore
    Explanation: Creates a .gitignore file to specify which files and folders should
    not be tracked by Git (like node_modules, __pycache__, etc.).

32. ADD ALL FILES TO GIT
    Command: git add .
    Explanation: Stages all files in the current directory for commit, preparing them
    to be saved to the git repository.

33. MAKE INITIAL COMMIT
    Command: git commit -m "Initial commit: Hospital Management System with Django backend and React frontend"
    Explanation: Creates the first commit with all the staged files, saving the current
    state of the project with a descriptive message.

34. SET MAIN BRANCH
    Command: git branch -M main
    Explanation: Renames the default branch to 'main' (modern convention instead of 'master').

35. PUSH TO GITHUB
    Command: git push -u origin main
    Explanation: Pushes the local commits to the remote GitHub repository. The -u flag
    sets up tracking so future pushes can be done with just 'git push'.

================================================================================
TROUBLESHOOTING COMMON ISSUES
================================================================================

1. Virtual Environment Not Activated:
   - Ensure you see (venv) in your terminal prompt
   - Run: venv\Scripts\Activate.ps1 (Windows PowerShell)

2. Port Already in Use:
   - Django: python manage.py runserver 8001 (different port)
   - React: npm start -- --port 3001 (different port)

3. Git Authentication Issues:
   - Use Personal Access Token instead of password
   - Set up SSH keys for authentication

4. Node Modules Issues:
   - Delete node_modules folder and package-lock.json
   - Run: npm install

5. Python Dependencies Issues:
   - Delete venv folder
   - Recreate virtual environment
   - Run: pip install -r requirements.txt

6. Git Submodule Issues (Frontend):
   - If you get "fatal: in unpopulated submodule 'frontend'" error:
   Commands to fix:
   git rm --cached frontend
   cd frontend
   Remove-Item -Recurse -Force .git (Windows PowerShell)
   cd ..
   git add frontend/
   git commit -m "Fix frontend: remove submodule and add as regular files"
   Explanation: This happens when React app creates its own git repository.
   These commands remove the submodule reference and add frontend as regular files.

7. Remove Git from Frontend Folder (Prevention):
   - After creating React app, immediately remove any .git folder:
   Command: cd fronmtend && Remove-Item -Recurse -Force .git (Windows PowerShell)
   Explanationn: Prevents frontend from becoming a git submodule and ensures
   git is only managed at the project root level.
